# PLAN-v0.1 — closing the loop v0 opened

**Status:** accepted plan for the v0.1 wave. Successor to `agent_docs/PLAN-paper-pptx.md`
(v0, complete). Grounded in the post-v0 four-angle gap review (production-reference mining,
self-declared scope cuts, ~30 public-API-only probes, and an end-to-end simulated template
job), with maintainer corrections folded in. "v0.1" names this plan wave; the distribution
version bump is decided at release time.

**The organizing insight, and the frame for everything below:** v0 built the *read half* of a
closed loop and stopped before the *write half*. It ships provenance-bearing inspection and
content-hash anchors that CONVENTIONS pins for "every targeting API" — and zero APIs that
consume an anchor. The most common brownfield deck edit (change this text, keep its
formatting) has no safe path. v0.1 exists to close that loop, and to fix the integrity gaps
the review proved by execution before adding anything new.

All v0 ground rules carry forward unchanged: CONVENTIONS.md is law; every change is additive;
upstream pytest + behave stay green on every PR; refusal atomicity and save→reopen on every
mutating API and test; fixture-first ("no fix without a fixture"); signatures land as an
API-PROPOSAL.md amendment before implementation, exactly like PR-0.

---

## Phase 0 — Integrity wave (nothing else starts until this merges)

v0.1 separates *defects in shipped behavior* from *missing capabilities*. Phase 0 is the
defect wave: four items, each probe-proven in the review, each landing fixture-first.

**0.1 — Sections dangling-ID defect.** `Slides.delete`/`reorder`/`clone` ignore
`p14:sectionLst` (and `p:custShowLst`): deleting a slide leaves its id dangling in the section
list — the exact corruption class Phase 7 existed to eliminate, in an ID list `relint` does
not scan. Fix BOTH halves: (a) slide ops maintain the lists (delete removes the sldId entry
from its section; reorder is id-keyed so it needs only a scan-verify; clone enrolls the copy
in the source's section, after its sldId); (b) the oracle learns to see the failure class —
extend `tests/paper/relint.py` (or a sibling `idlists.py` scan) to verify every id referenced
by `p14:sectionLst`/`p:custShowLst` exists in `p:sldIdLst`, run over every fixture and every
slide-op output. Never patch (a) without (b): the scan is what makes the corruption class
structurally impossible rather than individually fixed.

**0.2 / 0.3 — `inspect_text` silent omissions (table cells; grouped shapes).** Silence
violates CONVENTIONS §1.5. Resolution: **implement** text visibility — blocks from group
shapes via recursive `p:grpSp` descent (with a depth guard against pathological nesting) and
from table cells (`a:tbl` → row-major cell text bodies), with block anchors extended so a
block's position is stable under both (anchor addition is additive; pinned in the Phase 1
amendment since Phase 1 consumes them). Where a region's text is found but its *effective
values* remain unresolvable in v0.1 (table-cell runs inherit through table styles, a
different chain), the payload says so per-block — a typed, counted blind region, never an
absence. `effective_font` on a grouped `p:sp` run resolves normally (same chain); table-cell
runs keep their typed refusal until the table-style walk lands (backlog).

**0.4 — Wire `normalize_autofit` to the Phase 4 resolver.** Its own refusal message says
"resolve effective values before normalizing"; Phase 4 later built exactly that resolver in
the same package, and the organs were never connected. `normalize_autofit` gains
`resolve: bool = False` (keyword-only; default preserves v0 behavior exactly, per §1.1):
when set, unresolvable local sizes/spacing resolve through the effective walk before
freezing; what the walk cannot resolve still refuses, with provenance in the message. The
template-placeholder case — the reference's stated primary autofit use case — must pass.

**Phase 0 fixtures (before their fixes):** a sectioned deck — bootstrap self-generated by XML
injection, honestly labeled, plus a real-PowerPoint request (FIXTURE-REQUESTS.md R8, added
with this plan: sections are exactly the feature LibreOffice authoring may not faithfully
produce, so LO round-tripping is not an acceptable stand-in); a deck with tables inside
groups; a nested-group deck (≥3 levels, exercising the depth guard); an autofit placeholder
deck whose sizes resolve only through layout/master.

**Phase 0 also lands the walkthrough eval** (see "Definition of done"): the QBR simulation
frozen as `tests/paper/test_walkthrough_qbr.py`, with its currently-impossible steps encoded
as strict xfails that must flip as v0.1 phases land — the same forcing mechanism as the PR-0
stubs.

## Phase 1 — The write half (one wave, one shared design decision)

Anchored text replacement, shape surgery, and generic by-name addressing ship together
because they share the question the API-PROPOSAL amendment must answer *once, first*: **what
does an anchor-consuming write API look like?** The amendment pins: the call shape (method on
a span/target object vs. standalone function taking an anchor — recommendation: functions
beside their producers in `pptx.inspect`'s sibling namespace, since anchors are cross-part
addresses that no single proxy owns); staleness semantics — when `content_hash` mismatches,
**refuse** (typed, naming expected vs. found hash), never silently re-find; an explicit
`refind(anchor)` helper is the opt-in recovery path (fail loudly; the caller decides).
Whether stale-anchor refusal is a new `PaperRefusal` subclass or a `TargetNotFoundError`
message contract is settled in the amendment, not ad hoc.

**1.1 — Formatting-preserving text replace.** Pinned semantics (adopted from the review
feedback; the run-preservation doctrine originates in the reference's template-editing
notes — there is no docx-side code in this repo to mine, only the algorithm): split runs at
match boundaries; boundary fragments inherit their source run's `rPr` verbatim; runs the
match does not touch stay byte-identical; matches spanning run boundaries are handled (that
is the hard case and the reason this is an organ, not a helper). Scope forms: by anchor +
literal text, and deck-wide literal find/replace built on the same core. The §4 invariant
finally gets its subject: `replace(x→y)` then `replace(y→x)` restores visible text AND
formatting, byte-compared. Whitespace is content — the kernel's trailing-space trap pair is
a required fixture here too.

**1.2 — Shape surgery.** `delete` (with relationship cleanup — the dangling-rel bug class
found and fixed inside `replace_image` during v0 hardening defines the required test: global
rel scan after every delete, plus the shared-relationship case where a sibling shape still
references the same image part); z-order (`move` within the spTree — slides got reorder,
shapes didn't); copy-shape-to-another-slide (same-deck; relationship policy reusing the
Phase 7 machinery and refusal classes).

**1.3 — Generic by-name addressing.** `chart_by_name`'s contract (TargetNotFound /
AmbiguousTarget, never first-match) generalized: `shape_by_name`, `picture_by_name`,
`table_by_name` — same refusals, group-aware (finds named shapes inside groups, consistent
with 0.3's visibility rule). Removes the asymmetry v0's own API-PROPOSAL example tripped on.

## Phase 2 — Manifest and the evidenced extensions

**2.1 — Structural deck manifest.** The reference's single most-used helper
(`pptx_manifest`), as a typed, schema-versioned payload beside `inspect_text`: slide count,
page size, layout/master names, per-shape id/name/type/placeholder-idx/bbox/z-order, table
dimensions, chart type + series names, image content-type/natural-px/displayed size, autofit
state, notes presence. Deterministic, goldened per CONVENTIONS §2, group-recursive per 0.3.

**2.2 — Effective-walk extension.** Bold/italic/underline, alignment, line spacing for text;
**shape fill and line color** with the same provenance chain (the review's probe: v0 resolves
a run's theme color but not the rectangle's behind it). Same honesty rule: unresolved is
reported, never guessed.

**2.3 — Cross-format image replace.** Lift the extension-mismatch refusal by doing the work
it deferred: content-type + `[Content_Types].xml` handling on format change, partname
extension rewrite, same geometry/crop preservation, same shared-relationship safety. The v0
refusal remains the behavior when `allow_format_change=False` (the default stays refusal —
additive).

**2.4 — Workbook-less chart update.** `lo_chart_notes.pptx` is a standing red test: a chart
our own frozen corpus contains that `replace_data_safe` refuses today (no embedded workbook —
the LibreOffice/Google-authored shape). Support chart-XML-only data replacement with an
explicit no-workbook path; the refusal stays for chart *types* outside the validated family.

**2.5 — Fields and footers** (kept from the review over the feedback's shorter list — the
walkthrough evidence is direct: the QBR job had to fake page numbers as static text).
`a:fld` (slidenum/datetime) creation and recognition, and `p:hf` semantics on
slides/layouts/masters. Inspection reports fields distinctly from literal text (0.2/0.3's
payload already carries the distinction).

## Demand triggers (explicitly NOT phases)

Each converts a "bigger bet" into a wait-for-pull entry. Write the organ when the trigger
fires, not before:

  *Trigger: first real task requiring cross-deck import.* (Clone machinery is ~70% of it;
  the missing 30% — theme/master reconciliation — is the actual organ.)
- **4:3 ↔ 16:9 migration with content rescale.** *Trigger: first template-modernization task.*
- **Transitions/animations, read + strip.** *Trigger: first sanitize request that hits an
  animated deck.*
- **Table-style effective walk** (unlocks table-cell `effective_font`). *Trigger: first task
  needing table typography decisions, or 2.1 adoption feedback.*

## The walkthrough eval (v0.1's definition-of-done addition)

The gap review found what it found by *simulating the real job after v0 was declared done*.
v0.1 makes that simulation a permanent, deterministic test: `tests/paper/test_walkthrough_qbr.py`
executes the canonical template job end-to-end (clone template sections, retitle preserving
formatting, update the chart with new data, swap an image, write notes, normalize autofit,
footer/page-number pass, narrow-save) against the gauntlet fixture, asserting through the
contract harness at each step and LO-smoking the final artifact. Steps v0.1 hasn't shipped
yet are strict xfails (the PR-0 stub mechanism): the suite fails the moment a capability
lands without its walkthrough step flipping, and the walkthrough grows a step whenever a new
organ ships. **v0.1 is not done while any Phase 0–2 step in the walkthrough is an xfail.**

## Definition of done (per phase, unchanged + additions)

Everything from CONVENTIONS §9, plus: the Phase 0 integrity scans run over every fixture and
every organ output in the suite; the walkthrough eval's steps for the phase are flipped from
xfail to passing; PAPER.md entry written; API-PROPOSAL.md amended before any signature ships.

Order: 0 → 1 (gated on the anchor-write amendment) → 2. Phases 1 and 2 may interleave at the
organ level once Phase 0 has merged, except that 2.1's manifest schema waits for 0.2/0.3's
visibility rules (its payload must be born visibility-complete).
