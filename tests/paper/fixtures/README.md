# Fixture corpus

Frozen test corpus for the paper test suite, per `agent_docs/CONVENTIONS.md` §4. Read this
before adding, changing, or trusting any file here.

## The frozen-corpus rule

Every fixture's SHA-256 is pinned in `MANIFEST.sha256`
(`tests/paper/test_fixture_corpus.py` fails on any mismatch, unlisted fixture, missing or
schema-invalid sidecar, or orphan sidecar). Fixtures are **never regenerated by code under
test**. Changing or adding a fixture means, in one reviewed PR:

1. the fixture file itself,
2. its sidecar (`<stem>.json`, schema below),
3. a regenerated `MANIFEST.sha256`:
   `cd tests/paper/fixtures && find . -name "*.pptx" | sed 's|^\./||' | sort | while read f; do shasum -a 256 "$f"; done > MANIFEST.sha256`

## Provenance buckets (directory = who actually wrote the bytes)

| Directory | Meaning |
| --- | --- |
| `office_authored/` | Authored in desktop Microsoft PowerPoint by a human (none yet — see `FIXTURE-REQUESTS.md`) |
| `google_export/` | Exported from Google Slides (none yet — see `FIXTURE-REQUESTS.md`) |
| `libreoffice_export/` | Final bytes written by LibreOffice Impress (here: headless round-trips of self-generated decks; the sidecar `provenance.notes` says exactly that) |
| `other_producers/` | Any other real-world producer |
| `self_generated/` | Built by `tests/paper/_authoring/build_fixtures.py` with this repo's own code |

Provenance honesty is non-negotiable: a file's bucket and sidecar describe how its bytes were
actually produced, never how we wish they had been. Code that only passes on self-generated
files is untested — that is why the real-Office requests in `FIXTURE-REQUESTS.md` exist.

## Taxonomy coverage

| Taxonomy class | Fixtures |
| --- | --- |
| minimal-clean | `minimal_clean`, `lo_minimal_clean` |
| feature-isolated | `branded_template` (inherited size/font), `clrmap_remap`, `chart_notes` (clone fixture), `shared_media`, `autofit_none` / `autofit_normal` / `autofit_shape`, `whitespace_trailing_a`/`_b` (Phase 5 trap pair), plus the `lo_*` producer variants |
| gauntlet | `gauntlet` |
| corrupt-by-construction | `corrupt_dangling_sldid` (negative tests only) |
| large (perf smoke) | `large_smoke` |

A fixture is corrupt-by-construction iff any sidecar `features` entry starts with `"corrupt"`;
`tests.paper.corpus.is_corrupt_fixture()` is the single source of that rule. Corrupt fixtures
are never used as mutation targets.

## Sidecar schema (pinned by CONVENTIONS §4 — exactly these keys)

```json
{
  "fixture": "example.pptx",
  "provenance": {"app": "…", "version": "…", "notes": "…"},
  "features": ["…"],
  "ground_truth": {"…feature-keyed expected values…": "…"},
  "verified_by": "human name",
  "date": "YYYY-MM-DD"
}
```

Bootstrap sidecars carry `"verified_by": "UNVERIFIED - …"` because their ground truth was
machine-derived by direct XML inspection at authoring time and no human has verified it yet.
Where a claim is mechanically checkable, `test_fixture_corpus.py` re-verifies it against the
actual bytes on every run. Human verification (especially of the `branded_template`
effective-value expectations, which need desktop PowerPoint) is tracked in
`FIXTURE-REQUESTS.md`; verifying a sidecar means replacing the `verified_by` marker with your
name and the verification date.

## Golden-file policy

No golden files exist yet (the first arrive with Phase 4 inspection determinism tests). When
they do: goldens update only via an explicit, documented command, and golden diffs are
human-reviewed in the PR. "No fix without a fixture": every bug found downstream becomes a
frozen fixture plus a failing test before its fix merges.
